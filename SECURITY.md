# Security Policy

## Supported Versions

The Bible In Music is currently in its initial launch phase. We're committed to keeping the platform secure for all users.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x (Current) | :white_check_mark: |

As we're in early release, all users should be on the latest version of the platform.

## Reporting a Vulnerability

We take the security of The Bible In Music platform seriously. This includes protecting our users' data, payment information processed through Stripe, and ensuring uninterrupted access to Biblical content.

### How to Report

If you discover a security vulnerability, please report it by:

**Email:** info@thebibleinmusic.org

**Please include:**
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fixes (if applicable)

### What to Expect

- **Initial Response:** Within 7 days
- **Status Updates:** Every 7-14 days until resolved
- **Timeline:** We aim to address critical vulnerabilities within 30-60 days depending on severity

### Our Commitment

- We'll acknowledge receipt of your report promptly
- We'll keep you informed throughout the investigation
- If accepted, we'll work on a fix and notify you when it's deployed
- If declined, we'll explain our reasoning

**Please do not publicly disclose the vulnerability until we've had a chance to address it.**

Thank you for helping keep The Bible In Music safe for our community!
